ASTRI’s Sensor & Internet-of-Things (IoT) Gateway and Management Solution

 

Introduction

  • Technology Trend and Migration

Future Internet consists of billions of smart devices and sensors, which enable different smart applications like power and energy management, building management, home automation, healthcare, smart transport applications etc. Besides, there is a technology trend and migration for current small-scale, localized and isolated into future large-scale, ubiquitous and IP-enabled sensor, IoT and M2M applications and deployment by many on-going standard development and efforts.  

Challenges for the Migration

  • Connectivity
    • Large-scale, ubiquitous and data-intensive deployment, which involves huge amount of sensors, devices, and data.
    • Sensors and devices should be IPv6-connected, and accessible via the Internet with standard web services and protocols.
    • Gateway should be needed for connecting the constrained COAP-based 6LoWPAN sensor networks into the core IP network, with HTTP-COAP proxy and caching functionalities.
    • Gateway with standard web interfaces should also be needed for providing the embedded web services and connectivity.
  • Management
    • Multi-vendor sensor and device deployment, which involves various vendor-specific & proprietary sensors, devices and systems.
    • Multiple standards and protocols Integration like IEEE 802.15.4 6LoWPAN and Zigbee IP sensor standard, IETF CORE, ETSI M2M, 3GPP MTC, BACnet, Lonwork, Modbus etc.
    • Multi-vendor gateway and management platform should be needed to support different sensors and devices with different standard and protocols.
  • Application Enabling
    • Gateway with standard data interfacing should be needed for multiple data collaboration and management, which is necessary for enabling different sensor, IoT and M2M applications in web platform like smart home, smart building management system, healthcare etc.

ASTRI's Solution provides

  • Generic Sensor Gateway Platform (GSGP)
    • Multi-vendor adaptation and extensible platform
      • Allow rapid customization and adaptation using software adaptors for multi-vendor, multi-standard sensors, and devices integration and support.
      • Allow simplified, scalable and IPv6-connected deployment with common gateway platform.
      • Support various Contiki-OS &Tiny OS enabled, IEEE802.15.4, 6LoWPAN, Zigbee IP, BACnet, Lonwork, Modbus, vendor-specific sensors and devices etc.
    • Standard Webinterfaces for Data Interfacing and Control
      • Support standard REST/HTTP/COAP interface, XML and JSON data format for application enabling.
    • Customized Interfaces and Software Adaptors
      • Allow rapid customization, system integration and expansion for vendors’ existing systems.
  • Web-based Management System
    • Sensor, Device and Gateway Management
      • Support multi-vendor multi-standard sensor, device, and gateway management.
      • Provide inventory record, configuration, status, alarm, performance, security and access control & redundancy etc.
      • Support web-based user Interface.
    • Data Management
      • Support standard REST/HTTP/COAP interface and data format (e.g. XML, JSON) for cost-effective data adaptation, collaboration and mining.

ASTRI's Solution benefits

  • Sensor and Device Vendors, and System Integrators
    • Provides easy and fast system integration and expansion, enables migration to large-scale sensor, IoT, and M2M systems and applications.
    • Provides customizable management system for sensors, devices and gateway.
    • Enables efficient data management and mining for potential applications.
    • Offers complete end-to-end solution (sensors, devices, gateway, management system, applications) to customers.
  • Application Developers and Users
    • Provides standard data interfacing, collaboration and management, to enable different sensor, IoT and M2M applications like smart home, smart building management system, healthcare etc.
